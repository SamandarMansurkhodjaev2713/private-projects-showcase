## Русский

**Что это:** 3d-landing - публичный Three.js/WebGL landing с интерактивной 3D-сценой, визуальной подачей, motion и performance-aware frontend реализацией.

**Какую проблему решает:** многие портфолио-проекты выглядят как статичные страницы. Этот проект показывает, что я могу делать frontend, который не только работает, но и производит впечатление: сцена, движение, адаптивность, визуальный ритм и аккуратная презентация.

**Стек:** Three.js, WebGL, Vite, JavaScript/TypeScript direction, animation loop, responsive layout, GitHub Pages deployment.

**Архитектура:** Vite отвечает за dev/build pipeline, Three.js scene отделена от layout, animation loop управляет движением, UI слой держит контент и адаптивность, а deployment через GitHub Pages дает быстрый live proof.

**Почему именно так:** 3D frontend требует контроля над производительностью и визуальным состоянием. Разделение scene, animation и UI делает проект понятнее, легче оптимизировать и проще показывать как доказательство визуального инженерного уровня.

**Уникальность и сильные стороны:** visual engineering, интерактивность, live demo, WebGL, product presentation polish и способность делать интерфейсы, которые запоминаются.

**Что доказывает работодателю:** не только backend/system skills, но и сильный frontend taste: motion, композиция, responsiveness, performance и презентация.

**Что можно показать:** [live demo](https://samandarmansurkhodjaev2713.github.io/3d-landing/), [public repo](https://github.com/SamandarMansurkhodjaev2713/3d-landing), карточку и screenshot/video.

## English

**What it is:** 3d-landing is a public Three.js/WebGL landing with an interactive 3D scene, visual storytelling, motion and performance-aware frontend implementation.

**Problem it solves:** many portfolio projects look like static pages. This project shows that I can build frontend that not only works, but creates an impression: scene, motion, responsiveness, visual rhythm and polished presentation.

**Stack:** Three.js, WebGL, Vite, JavaScript/TypeScript direction, animation loop, responsive layout and GitHub Pages deployment.

**Architecture:** Vite owns the dev/build pipeline, the Three.js scene is separated from layout, the animation loop controls motion, the UI layer handles content/responsiveness, and GitHub Pages deployment provides fast live proof.

**Why this architecture:** 3D frontend requires control over performance and visual state. Separating scene, animation and UI makes the project easier to understand, optimize and present as visual engineering proof.

**Unique strengths:** visual engineering, interactivity, live demo, WebGL, product presentation polish and the ability to create interfaces that are memorable.

**Employer signal:** not only backend/system skills, but strong frontend taste: motion, composition, responsiveness, performance and presentation.

**Safe proof:** [live demo](https://samandarmansurkhodjaev2713.github.io/3d-landing/), [public repo](https://github.com/SamandarMansurkhodjaev2713/3d-landing), project card and screenshot/video.
